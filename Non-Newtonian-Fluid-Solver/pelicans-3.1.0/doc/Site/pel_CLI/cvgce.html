NAME

cvgce - extraction of error norms from save_*.gene text files

SYNOPSIS

pel cvgce [-help|-man]

pel cvgce [-v] <norm_saving_names>

DESCRIPTION

The convergence properties of numerical schemes when refining the time step and the mesh size may be tested using the classes FE_Launcher and FE_ComparatorWithAnalytic. In this context and when the calculation results are saved with PEL_TICwriter (in the ``text'' mode), a sequence of save_*.gene files are generated. Each save_*.gene corresponds to a given time step, to a given mesh size (specified on creation of FE_Launcher), and stores at all cycles the error norms that where requested from FE_ComparatorWithAnalytic.

Given such a sequence of save_*.gene files, pel cvgce extracts all the values of the error norms, reorganizes these data to stress the dependencies with the time step and the mesh size (for a given mesh size, dependence with the time step ; for a given time step, dependence with the mesh size), and finally produces multicolumn ascii files that may be used for graphic postprocessing by usual tools (eg gnuplot or xmgrace).

OPTIONS

-h, -help

Print a brief help message and exit.

-m, -man

Print the manual page and exit.

-v, -verbose

Execute verbosely (recommended).

ARGUMENTS

norm_saving_names

Items occuring in the entry of keyword norm_saving_names in the Hierarchical Data Structure used to create the FE_ComparatorWithAnalytic object.

EXAMPLE

pel cvgce -v XLDU XHDU

Extract from all the save_*.gene ascii files of the current directory the entries XLDU and XHDU, and produce the following files:

XLDU_h.txt : for all time steps, XLDU as a function of the mesh size

XLDU_dt.txt : for all mesh sizes, XLDU as a function of the time step

XHDU_h.txt : for all time steps, XHDU as a function of the mesh size

XHDU_dt.txt : for all mesh sizes, XHDU as a function of the time step

The save_*.gene files have been produced with an application using a FE_ComparatorWithAnalytic which has been created using a PELICANS Hierarchical Data Structure. This latter must be such that "XLDU" and "XLDU" occur in its entry of keyword norm_saving_names.
